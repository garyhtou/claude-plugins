# Research

The longer-form, fully-cited research behind the `high-fidelity-web` plugin. The
skills' `references/` files are the distilled, instructional versions of this; these
documents keep the full reasoning, the numbers, and the sources, so the plugin is
self-contained and its claims are checkable.

Built with AI under the guidance of Gary Tou. Each file ends with a Sources section.

| File | What it covers |
|---|---|
| `01-design-principles.md` | Taste and craft: easing, timing, springs, typography, color, depth without WebGL, anti-patterns. |
| `02-techniques-and-implementations.md` | How the effects are built (scroll, smooth scroll, WebGL heroes, transitions, micro-interactions, perf), with code and anatomies of famous effects. |
| `03-tools-and-libraries.md` | The "when to use what" map of the animation/3D/web tooling landscape. |
| `04-workflow-and-process.md` | How design engineers take an idea to a polished site: the end-to-end process and the agent-adapted workflow. |
| `05-live-site-inspection.md` | Firsthand inspection of a real high-fidelity site (design tokens, decorative WebGL, scroll approach). |
| `06-reliability-and-gotchas.md` | Production failure modes and fixes: scroll/reveal triggers, pinned-scroll and scroll-restoration bugs, rAF lifecycle, WebGL/headless, layout stability, tooltip and hit-target details. |

Facts in `03` (bundle sizes, library versions, licensing, browser support) move
fast; each is dated, re-verify before relying on a specific number.
